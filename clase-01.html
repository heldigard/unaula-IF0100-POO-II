Teoría

Qué es C# y por qué usarlo

C# es un lenguaje moderno, orientado a objetos y type-safe. Se utiliza para crear aplicaciones web, escritorio, servicios en la nube, videojuegos y sistemas empresariales.

Ventajas clave

  • Multiplataforma (Windows, Linux, macOS).
  • Alto rendimiento y compilación con JIT.
  • Ecosistema .NET amplio y estable.

Ecosistema .NET y arquitectura

.NET 8 unifica el runtime para web, desktop, mobile, cloud y gaming. Antes existían plataformas separadas (Framework/Core/Xamarin); ahora se comparte el mismo runtime y librerías base.

┌─────────────────────────────────────────┐
│            TU APLICACIÓN C#             │
├─────────────────────────────────────────┤
│     BCL (Base Class Library)            │
│   System.String, Collections, IO, Data  │
├─────────────────────────────────────────┤
│     CLR (Common Language Runtime)       │
│  JIT | GC | Type Safety | Exceptions    │
├─────────────────────────────────────────┤
│        Windows / Linux / macOS          │
└─────────────────────────────────────────┘

Componentes clave

  • CLR: Ejecuta programas, maneja memoria y provee servicios como thread management.
  • BCL: Librería estándar con String, Collections, IO, Data, etc.
  • IL: Código intermedio que el CLR convierte a código máquina.
  • GC: Gestión automática de memoria, previene memory leaks.
  • JIT: Compila IL a código máquina cuando se ejecuta el programa.

¿Por qué importa esta arquitectura?

La arquitectura en capas permite que el mismo código C# funcione en diferentes sistemas operativos sin cambios. El CLR abstrae las diferencias de plataforma.

Error común: Creer que C# es solo para Windows. Desde .NET Core, C# es verdaderamente multiplataforma.

Mini-chequeo: ¿Puedes explicar qué hace el CLR? ¿En qué se diferencia IL del código máquina?

Estructura básica de un programa C#

Los elementos esenciales son using, namespace, class y el método Main como punto de entrada.

Namespace: Es como un "apellido" para las clases. Dos clases pueden llamarse Program si están en namespaces diferentes. Similar a los paquetes en Java.

using System;

namespace MiApp
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hola, C#");
        }
    }
}

Tipos de datos: value vs reference

En C#, los tipos se dividen en dos categorías que afectan cómo se almacenan y copian los datos.

Value types (almacenados en stack)

Contienen directamente el valor. Al copiar, se crea una copia independiente. Incluyen: int, double, bool, struct.

int edad = 25;
int edad2 = edad;
edad2 = 30;
Console.WriteLine(edad);  // 25 (no cambió)

Reference types (almacenados en heap)

Contienen una referencia a un objeto. Al copiar, se comparte la referencia. Incluyen: class, string, array.

string nombre = "Juan";
string nombre2 = nombre;
nombre2 = "Maria";
Console.WriteLine(nombre); // Juan (string es inmutable)
Nota: string es reference type, pero es inmutable; al reasignar se crea un nuevo objeto. Para comportamientos mutables, usar StringBuilder.
Error común: Creer que modificar una variable de referencia afecta a todas las variables que apuntan al mismo objeto. Depende de si el tipo es mutable o inmutable.

Mini-chequeo: ¿Cuál es la diferencia fundamental entre value types y reference types al asignar una variable a otra?

Nullable, strings y patrones

Nullable (tipos que pueden ser null)

Por defecto, los value types (int, double, bool) no pueden ser null. El operador ? permite que acepten null.

¿Por qué usarlo? Útil cuando un valor puede estar ausente (ej: edad desconocida de un usuario). El operador ?? (null-coalescing) proporciona un valor por defecto.

int? edad = null;         // puede ser null
int edadDefault = edad ?? 18;  // 18 si edad es null
Error común: No validar null antes de usar un nullable. Usar if (edad.HasValue) o ??.

Pattern matching (coincidencia de patrones)

Forma moderna y legible de expresar condicionales complejas. El switch expression devuelve un valor directamente.

¿Por qué usarlo? Código más limpio que múltiples if-else, y el compilador verifica que todos los casos estén cubiertos.

string categoria = edadDefault switch
{
    < 13 => "Niño",
    < 20 => "Adolescente",
    < 65 => "Adulto",
    _ => "Mayor"  // "_" es el default (cualquier otro caso)
};

Mini-chequeo: ¿Qué pasa si edadDefault es 70? ¿Y si es -5?

Primary constructors (constructores primarios)

Sintaxis simplificada para clases que solo necesitan recibir parámetros en el constructor.

¿Por qué usarlo? Reduce código boilerplate cuando la clase solo almacena parámetros.

public class Persona(string nombre, int edad)
{
    public string Info => $"{nombre} ({edad} años)";
}
Nota: Los parámetros del primary constructor están disponibles en toda la clase, similar a campos privados.

Collection expressions

Sintaxis moderna para crear colecciones sin repetir new[] o new List<T>().

int[] pares = [2, 4, 6];
int[] impares = [1, 3, 5];
int[] all = [..pares, ..impares];  // spread operator

Ejemplo: LINQ para consultas funcionales

LINQ permite consultar colecciones (listas, arrays, bases de datos) de forma declarativa, similar a SQL pero dentro de C#.

¿Por qué usarlo? Código más legible que loops foreach anidados, menos propenso a errores, y permite encadenar operaciones.

Filtrar y transformar lista

List nums = new() { 1, 2, 3, 4, 5 };
var pares = nums.Where(n => n % 2 == 0)     // filtrar pares
                .Select(n => n * 2)          // duplicar
                .ToList();                   // convertir a lista
// Resultado: [4, 8]

Beneficios

  • Código declarativo y legible.
  • Integrado en el lenguaje.
  • Optimizado por el compilador.
Error común: Creer que LINQ siempre crea una lista nueva. Algunos métodos como Where retornan un iterador lazy (se ejecutan solo cuando se enumera).

Mini-chequeo: ¿Qué hace Where? ¿Qué pasa si llamas ToList() antes de Select?

Ciclo de ejecución en C#

Entender cómo se ejecuta un programa C# ayuda a diagnosticar problemas y optimizar rendimiento.

1) Código C# (.cs) - Código fuente legible por humanos
2) Compilación -> IL (.dll/.exe) - Código intermedio (CIL)
3) CLR (JIT) -> código máquina - Compilación Just-In-Time
4) Ejecución en Windows/Linux/macOS - Código nativo del CPU

Fases clave

  • Compilación a IL: El compilador C# convierte código fuente a IL, independiente de la plataforma.
  • JIT compilation: El CLR compila IL a código máquina nativo durante la ejecución, específico para cada CPU/SO.
  • NGen: Opcionalmente, IL puede pre-compilarse a nativo para reducir el tiempo de inicio.

¿Por qué este diseño?

La compilación en dos pasos permite que el mismo código IL funcione en diferentes plataformas sin recompilar. El JIT optimiza según el hardware específico.

Nota: El código IL puede desensamblarse con herramientas como ILSpy o dotPeek, útil para aprender cómo funciona el código compilado.

Depuración básica (10 min)

Debug (depuración) es ejecutar el programa controladamente para inspeccionar valores y encontrar errores.

Breakpoint es un marcador que pausa la ejecución en una línea específica. Presiona F9 en la línea deseada.

AtajoAcciónDescripción
F5Iniciar debugComienza ejecución con debugger
F9Toggle breakpointAñade/quita breakpoint en línea actual
F10Step OverEjecuta línea actual sin entrar en funciones
F11Step IntoEntra en funciones llamadas
Shift+F11Step OutSale de función actual
Tip: Pasa el mouse sobre variables para ver sus valores durante el debug. Usa la ventana "Inmediato" (Ctrl+Alt+I) para evaluar expresiones.

Errores comunes y checklist

  • Olvidar using System; cuando se usa Console.
  • No validar entradas de usuario antes de convertir a numérico.
  • No configurar workloads correctos en Visual Studio.
  • Confundir value types con reference types al asignar.

Actividad: Ejercicio 1 - Calculadora de área (20 min)

Tipo: Opcional / práctica

Entregable: Aplicación de calculadora de área funcional

Tiempo estimado: 20 min

using System;

namespace CalculadoraArea
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("=== CALCULADORA ===");
            Console.Write("Base: ");
            double b = double.Parse(Console.ReadLine());
            Console.Write("Altura: ");
            double h = double.Parse(Console.ReadLine());
            double area = b * h;
            Console.WriteLine($"Área: {area:F2}");
        }
    }
}

Actividad: Ejercicio 2 - Sistema de calificaciones (30 min)

Tipo: Opcional / práctica

Relacionado con: Ninguno

Entregable: Sistema de calificaciones con validación

Tiempo estimado: 30 min

Criterios de éxito: Manejo de entrada, validación, condicionales

Solicitar nombre y tres notas (0.0 - 5.0), calcular promedio y mostrar APROBADO/REPROBADO con validación.

  • Checklist: validación de entrada, cálculo de promedio, condición de aprobación (>=3.0), manejo de errores.
static double PedirNota(string etiqueta)
{
    double nota;
    while (true)
    {
        Console.Write($"{etiqueta} (0-5): ");
        if (double.TryParse(Console.ReadLine(), out nota))
        {
            if (nota >= 0 && nota <= 5) return nota;
            Console.WriteLine("Fuera de rango");
        }
        else
        {
            Console.WriteLine("No válido");
        }
    }
}
