Teoría

Qué es Python y por qué usarlo

Python es un lenguaje moderno, orientado a objetos y type-safe con ayuda de herramientas. Se utiliza para automatización, backend web, análisis de datos, IA y aplicaciones empresariales.

Ventajas clave

  • Sintaxis clara y legible, ideal para aprendizaje.
  • Ecosistema enorme de librerías (PyPI).
  • Portabilidad (Windows, Linux, macOS).

Ecosistema Python y arquitectura

Python ejecuta código a través de un intérprete (CPython en la mayoría de casos) y una biblioteca estándar amplia. El flujo típico incluye entornos virtuales, instalación de paquetes con pip y ejecución multiplataforma.

Componentes clave

  • Intérprete: Ejecuta el bytecode generado a partir de archivos .py.
  • Standard Library: Librerías incluidas para tareas comunes (IO, json, datetime, logging).
  • PyPI + pip: Repositorio y gestor de paquetes para dependencias externas.
  • Entornos virtuales: Aísla dependencias por proyecto (python -m venv).
  • Bytecode: Caché .pyc que acelera ejecuciones posteriores.

¿Por qué importa esta arquitectura?

El intérprete y la stdlib permiten que el mismo código Python funcione en diferentes sistemas operativos con mínimas diferencias.

Error común: asumir que Python siempre es lento. El rendimiento depende del algoritmo y de librerías optimizadas.

Tipos de datos: mutables vs inmutables

En Python todas las variables son referencias a objetos. Lo que cambia es si el objeto es mutable o inmutable.

Inmutables (no cambian en el lugar)

Incluyen int, float, str y tuple. Al modificar, se crea un nuevo objeto.

immutables.py
1
2
3
4
edad = 25
edad2 = edad
edad2 = 30
print(edad)  # 25 (no cambió)

Mutables (cambian en el lugar)

Incluyen list, dict y set. Dos variables pueden apuntar al mismo objeto.

mutables.py
1
2
3
4
nombres = ["Ana", "Luis"]
alias = nombres
alias.append("Sofía")
print(nombres)  # ['Ana', 'Luis', 'Sofía']
Nota: para evitar efectos colaterales, usa copias con list() o copy().
Error común: asumir que asignar una lista crea una copia independiente. En realidad comparte referencia.

Ejemplo: comprensiones y funciones de orden superior

Python ofrece list comprehensions y funciones como map y filter para transformar colecciones de forma declarativa.

¿Por qué usarlo? Código más legible que bucles anidados y fácil de mantener.

Filtrar y transformar lista

comprehensions.py
1
2
nums = [1, 2, 3, 4, 5]
pares = [n * 2 for n in nums if n % 2 == 0]

Beneficios

  • Código declarativo y legible.
  • Integrado en el lenguaje.
  • Se puede combinar con map, filter y sorted.
Error común: olvidar que las comprensiones crean una lista nueva; para grandes volúmenes usa generadores.

Ciclo de ejecución en Python

Entender cómo se ejecuta un programa Python ayuda a diagnosticar problemas y optimizar rendimiento.

Ciclo de Ejecución
1
2
3
1) Código Python (.py) - Código fuente legible por humanos
2) Bytecode (.pyc) - Compilación ligera por el intérprete
3) Ejecución por el intérprete en Windows/Linux/macOS

Fases clave

  • Compilación a bytecode: El intérprete genera .pyc para acelerar ejecuciones futuras.
  • Interpretación: CPython ejecuta instrucciones del bytecode.
  • Gestión de memoria: Recolección de basura y conteo de referencias.

¿Por qué este diseño?

Permite un balance entre portabilidad y rendimiento, con una curva de aprendizaje suave.

Nota: herramientas como pyenv ayudan a gestionar múltiples versiones de Python.

Depuración básica (10 min)

Debug (depuración) es ejecutar el programa controladamente para inspeccionar valores y encontrar errores en Python.

Breakpoint es un marcador que pausa la ejecución en una línea específica. Presiona F9 en la línea deseada en VS Code.

AtajoAcciónDescripción
F5Iniciar debugComienza ejecución con debugger
F9Toggle breakpointAñade/quita breakpoint en línea actual
F10Step OverEjecuta línea actual sin entrar en funciones
F11Step IntoEntra en funciones llamadas
Shift+F11Step OutSale de función actual
Tip: Pasa el mouse sobre variables para ver sus valores durante el debug. Usa la ventana "Inmediato" (Ctrl+Alt+I) para evaluar expresiones.

Errores comunes y checklist

  • Olvidar activar el entorno virtual antes de instalar paquetes.
  • Confundir listas mutables con copias independientes.
  • No validar entradas de usuario antes de convertir a numérico.
  • Ejecutar con una versión de Python distinta a la del curso.
