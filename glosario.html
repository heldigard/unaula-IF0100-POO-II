Glosario de Términos

Diccionario de acrónimos y conceptos del curso IF0100

  1. Inicio
  2. Glosario

POO: Programación Orientada a Objetos. Paradigma basado en el uso de objetos que interactúan entre sí.

TDD: Test-Driven Development. Desarrollo guiado por pruebas (escribir el test antes que el código).

BDD: Behavior-Driven Development. Desarrollo guiado por comportamiento (usando lenguaje Gherkin).

DDD: Domain-Driven Design. Diseño orientado al dominio (enfocarse en la lógica de negocio).

API: Application Programming Interface. Conjunto de reglas que permiten que dos aplicaciones se comuniquen.

ORM: Object-Relational Mapping. Técnica para mapear objetos de código a tablas de base de datos SQL.

JWT: JSON Web Token. Estándar para transmitir información de forma segura para autenticación.

HTMX: Librería que permite acceder a funciones AJAX y transiciones directamente desde HTML.

DTO: Data Transfer Object. Objetos simples usados para transportar datos entre capas (en Python usamos Pydantic).

MRO: Method Resolution Order. El orden en que Python busca métodos en jerarquías de herencia múltiple.

Conceptos de Arquitectura

Clean Architecture: Arquitectura limpia que busca la independencia de frameworks y bases de datos.

Patrón Repository: Abstracción que desacopla la lógica de negocio del acceso real a los datos (archivos o SQL).
