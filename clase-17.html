Teoria

Proyecto integrador: Sistema de Gestión Académica

El proyecto final integra POO avanzada, ADO.NET, DataSet, DataBinding, XML y pruebas unitarias en un sistema realista.

  • POO avanzada y patrones (Repository, Service, Unit of Work).
  • Persistencia con SQL Server y exportacion XML.
  • Interfaz grafica (Windows Forms / WPF).
  • Pruebas unitarias con xUnit/NUnit.

Repository Pattern (fragmento)

public interface IEstudianteRepository
{
    Estudiante ObtenerPorId(int id);
    IEnumerable<Estudiante> ObtenerTodos();
    void Agregar(Estudiante estudiante);
    void Actualizar(Estudiante estudiante);
    void Eliminar(int id);
}

Unit of Work (fragmento)

public interface IUnitOfWork : IDisposable
{
    IEstudianteRepository Estudiantes { get; }
    ICursoRepository Cursos { get; }
    INotaRepository Notas { get; }
    void Commit();
    void Rollback();
}

Practica / laboratorio

Módulo mínimo (checklist)

  • CRUD Estudiantes con validación.
  • CRUD Cursos con capacidad maxima.
  • Gestión de notas y calculo de promedios.
  • Reportes: top 10, estudiantes en riesgo.
  • Exportación XML.

Cronograma sugerido

SemanaActividad
1-2Modelo BD y entidades
3-4Repositorios y servicios
5-6UI (Windows Forms/WPF)
7Pruebas y corrección
8Documentación y demo

Checklist de calidad

  • Capas separadas (Core/Data/UI/Tests).
  • Sin SQL concatenado (parametros).
  • Testing con AAA y cobertura base.
  • README con pasos de ejecución.

Ejercicios

Diseño del modelo

Identificar Entities, relaciones y reglas de negocio del sistema academico.

Documentación

Crear README con instalación, requisitos y capturas del sistema.

Checklist de errores comunes

  • Mezclar logica de negocio en la UI.
  • No validar entradas de usuario.
  • No versionar scripts SQL.
  • No incluir pruebas automatizadas.

Referencias
