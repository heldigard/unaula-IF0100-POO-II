¿Por qué HTML5 Semántico?

HTML5 introdujo etiquetas con significado (semántica) en lugar de genéricas <div>. Esto permite:

  • Mejor SEO: Los motores de búsqueda entienden mejor tu contenido
  • Accesibilidad: Los lectores de pantalla navegan más fácilmente
  • Mantenibilidad: El código es más legible y auto-documentado
  • Consistencia: Estructura estándar en toda la web moderna

3. Bootstrap: Introducción

Framework CSS Más Popular

Bootstrap 5 es un framework CSS/JavaScript para desarrollo responsivo y mobile-first de proyectos web.

Ahorra tiempo Diseño responsivo Componentes listos Cross-browser Documentación extensa Personalizable

Ventajas de Bootstrap

  • No reinventar la rueda: Componentes pre-construidos y probados
  • Responsivo incluido: Sistema de grillas mobile-first
  • Compatible: Funciona en todos los navegadores modernos
  • Documentación extensa:Ejemplos y guías detalladas
  • Personalizable: variables Sass y temas personalizables

4. Sistema de Grillas Bootstrap

El Corazón de Bootstrap

Bootstrap usa un sistema de 12 columnas por fila con 6 breakpoints para diseño mobile-first.

Clase Breakpoint Típicamente
col- Extra small Móviles portrait
col-sm- Small (576px) Móviles landscape
col-md- Medium (768px) Tablets
col-lg- Large (992px) Laptops
col-xl- Extra large (1200px) Desktops
col-xxl- Extra extra large (1400px) Pantallas grandes

Ejemplo: col-md-6 = En tablets ocupa 6/12 (50%)

Práctica / Laboratorio (30 min)

Actividad: Landing Page con Bootstrap 5 (30 min)

Tipo: Opcional / práctica

Relacionado con: Ninguno

Entregable: Landing page responsive con Bootstrap 5

Tiempo estimado: 30 min

Criterios de éxito: Proyecto compila, Bootstrap cargando, página responsive en móvil y desktop

Objetivo: Crear una landing page responsive con Bootstrap 5.

Pasos en Visual Studio 2022

  1. Abrir Visual Studio 2022 y crear un proyecto ASP.NET Core Web App.
  2. En el Explorador de Soluciones, clic derecho en Pages > Agregar > Nuevo elemento > Razor Page.
  3. Nombre: Index en la carpeta Pages (sobrescribir el existente).
  4. Agregar Bootstrap 5 via CDN o NuGet en Pages/Shared/_Layout.cshtml.
  5. Ejecutar con F5 y probar la responsive con las herramientas de desarrollador (F12).
Checklist: Proyecto compila sin errores, Bootstrap cargando correctamente, página responsive en móvil y desktop.

Estructura HTML Básica con Bootstrap

<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>Mi Landing Page</title>
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css"
          rel="stylesheet" />
</head>
<body>
    <nav class="navbar navbar-expand-lg navbar-dark bg-primary">
        <div class="container">
            <a class="navbar-brand" href="#">MiMarca</a>
        </div>
    </nav>
    <div class="container my-5">
        <div class="row">
            <div class="col-12 col-md-6">
                <h1>Bienvenido</h1>
                <p class="lead">Descripción del producto.</p>
                <a class="btn btn-primary btn-lg" href="#">Llamada a la Acción</a>
            </div>
        </div>
    </div>
    <footer class="bg-dark text-white text-center py-3 mt-5">
        <p>&copy; 2026 Mi Landing Page</p>
    </footer>
    <script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/js/bootstrap.bundle.min.js"></script>
</body>
</html>
Checklist: Estructura Navbar implementada, grid system usado (col-12, col-md-6), botón CTA funcional, footer agregado, scripts Bootstrap cargando.

Ejercicios Prácticos

Actividad: Ejercicio 1 - Landing Page (20 min)

Tipo: Opcional / práctica

Relacionado con: Ninguno

Entregable: Landing page con hero, features, formulario y footer

Tiempo estimado: 20 min

Criterios de éxito: Hero implementada, grid responsive, formulario con validaciones, footer funcionando

Crear una landing page con:

  • Hero section con título y botón de acción
  • Features en 3 columnas (1 móvil, 2 tablet, 3 desktop)
  • Formulario de contacto con validación HTML5
  • Footer con enlaces
Checklist: Hero section implementada, grid responsive correctamente, formulario HTML5 con validaciones, footer funcionando.

Actividad: Ejercicio 2 - Dashboard de Estudiantes (25 min)

Tipo: Opcional / práctica

Relacionado con: Ninguno

Entregable: Dashboard con navbar, sidebar, cards, tabla y modal

Tiempo estimado: 25 min

Criterios de éxito: Navbar con dropdown, sidebar offcanvas, cards con datos, tabla con paginación, modal de confirmación

Crear un dashboard con:

  • Navbar con dropdown
  • Sidebar colapsable (offcanvas)
  • Cards con estadísticas (tales ejemplos, promedios)
  • Tabla de datos con paginación
  • Modal para confirmar eliminación
Checklist: Navbar con dropdown funcional, sidebar offcanvas operativo, cards mostrando datos, tabla con paginación, modal de confirmación funcionando.

Actividad: Ejercicio 3 - Página de Estudiantes (30 min)

Tipo: Opcional / práctica

Relacionado con: Ninguno

Entregable: Vista ListaEstudiantes con layout responsive y badges

Tiempo estimado: 30 min

Criterios de éxito: Layout responsive 1-2-3, cards con datos, badges de colores, botón de detalle, formulario de búsqueda

Crear Vista "ListaEstudiantes" con:

  • Layout responsivo (1 columna móvil, 2 tablet, 3 desktop)
  • Card de ejemplo con foto, nombre, carrera (badge), promedio
  • Badge de color según promedio: verde (>=3.5), amarillo (>=3.0), rojo (<3.0)
  • Botón "Ver detalle"
