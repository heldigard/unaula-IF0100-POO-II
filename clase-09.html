1. HTML5: Estructura Semántica

Más allá de

¿Por qué HTML5 Semántico?

HTML5 introdujo etiquetas con significado (semántica) en lugar de genéricas <div>. Esto permite:

  • Mejor SEO: Los motores de búsqueda entienden mejor tu contenido
  • Accesibilidad: Los lectores de pantalla navegan más fácilmente
  • Mantenibilidad: El código es más legible y auto-documentado
  • Consistencia: Estructura estándar en toda la web moderna

3. Bootstrap: Introducción

Framework CSS Más Popular

Bootstrap 5 es un framework CSS/JavaScript para desarrollo responsivo y mobile-first de proyectos web.

Ahorra tiempo Diseño responsivo Componentes listos Cross-browser Documentación extensa Personalizable

Ventajas de Bootstrap

  • No reinventar la rueda: Componentes pre-construidos y probados
  • Responsivo incluido: Sistema de grillas mobile-first
  • Compatible: Funciona en todos los navegadores modernos
  • Documentación extensa:Ejemplos y guías detalladas
  • Personalizable: variables Sass y temas personalizables

4. Sistema de Grillas Bootstrap

El Corazón de Bootstrap

Bootstrap usa un sistema de 12 columnas por fila con 6 breakpoints para diseño mobile-first.

Clase Breakpoint Típicamente
col- Extra small Móviles portrait
col-sm- Small (576px) Móviles landscape
col-md- Medium (768px) Tablets
col-lg- Large (992px) Laptops
col-xl- Extra large (1200px) Desktops
col-xxl- Extra extra large (1400px) Pantallas grandes

Ejemplo: col-md-6 = En tablets ocupa 6/12 (50%)

Utilidades Bootstrap

Spacing

m = margin, p = padding

t = top, b = bottom, s = start, e = end, x = horizontales, y = vertical

Valores 0-5: 0, 0.25rem, 0.5rem, 1rem, 1.5rem, 3rem

<div class="m-3">Margin 3 en dos lados</div>
<div class="mt-2 mb-4">Margin top 2, bottom 4</div>
<div class="px-5 py-3">Padding horizontales 5, vertical 3</div>
<div class="mx-auto">Centrado horizontales (auto)</div>

Accesibilidad - ARIA Labels

Importante para usuarios con discapacidades. ARIA = Accessible Rich Internet Applications

<!-- ARIA-LABEL - Descripción para elementos sin texto -->
<input type="search" aria-label="Buscar ejemplos">
<button aria-label="Cerrar ventana">&times;</button>

<!-- ARIA-LABELEDBY - Relacionar con otro elemento -->
<input type="text" id="nombre" aria-labelledby="lbl-nombre">
<label id="lbl-nombre">Nombre completo:</label>

<!-- ARIA-DESCRIBEDBY - Descripción adicional -->
<input type="password" aria-describedby="pwd-help">
<small id="pwd-help">Mínimo 8 caracteres, una mayúscula y un número</small>

<!-- ARIA-HIDDEN - Ocultar a lectores de pantalla -->
<span aria-hidden="true"></span>

<!-- ROLES ARIA - Semántica adicional -->
<nav role="navigation" aria-label="Menú principal">...</nav>
<main role="main">...</main>
<div role="alert" class="alert alert-danger">
    Operación fallida
</div>

Objetivo: Crear una landing page responsive con Bootstrap 5.

Pasos en Visual Studio 2022

  1. Abrir Visual Studio 2022 y crear un proyecto ASP.NET Core Web App.
  2. En el Explorador de Soluciones, clic derecho en Pages > Agregar > Nuevo elemento > Razor Page.
  3. Nombre: Index en la carpeta Pages (sobrescribir el existente).
  4. Agregar Bootstrap 5 via CDN o NuGet en Pages/Shared/_Layout.cshtml.
  5. Ejecutar con F5 y probar la responsive con las herramientas de desarrollador (F12).
Checklist: Proyecto compila sin errores, Bootstrap cargando correctamente, página responsive en móvil y desktop.

Resumen de la Clase

  • ADO.NET es la API de .NET para acceso a datos con SqlConnection, SqlCommand, SqlDataReader.
  • El patrón conectado mantiene conexión abierta durante lectura (SqlDataReader es forward-only).
  • Parameters previenen SQL Injection y manejan tipos de datos correctamente.
  • Transacciones (SqlTransaction) garantizan atomicidad: Commit confirma, Rollback revierte.
  • DELETE físico elimina filas; DELETE lógico marca como inactivo (soft delete).
  • Paginación con OFFSET/FETCH mejora rendimiento en datasets grandes.
  • SqlBulkCopy permite inserción masiva de miles de filas en segundos.
  • LINQ to SQL y Entity Framework son ORMs que abstraen SQL, pero ADO.NET directo es más rápido.
  • Connection String contiene servidor, base de datos, autenticación (Windows o SQL Server).
  • Using statements aseguran liberación de recursos (Dispose pattern) incluso con excepciones.
