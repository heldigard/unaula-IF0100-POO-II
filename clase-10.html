Teoría

Patrón MVC y responsabilidades

El patrón MVC (Model-View-Controller) separa la aplicación en tres componentes: el controlador orquesta el flujo, el modelo contiene reglas y datos, y la vista presenta HTML. La separación evita mezclar UI con lógica de negocio.

¿Por qué MVC?

MVC es uno de los patrones más usados en desarrollo web porque permite:

  • Separación de responsabilidades: Cada componente hace una cosa bien definida
  • Paralelización: Diseñadores y desarrolladores pueden trabajar simultáneamente
  • Testabilidad: Controllers y Models se pueden probar sin la vista
  • Mantenibilidad: Cambios en UI no afectan lógica de negocio
  • Reutilización: Models y Views se pueden reutilizar en diferentes contextos
Controller: recibe request, valida, decide flujo
Model: entidades, reglas, acceso a datos
View: presentación HTML/CSS (Razor)

Model Binding y ViewModels

El Model Binding de ASP.NET Core mapea datos HTTP a objetos C#. Los ViewModels separan el modelo de dominio de la UI e incluyen validaciones.

public class EstudianteViewModel
{
    [Required]
    [Display(Name = "Código")]
    public string Codigo { get; set; }

    [Required]
    public string Nombre { get; set; }

    public SelectList Carreras { get; set; }
}

Razor Pages y Tag Helpers

  • Razor Pages simplifica páginas con PageModel + .cshtml.
  • Tag Helpers generan HTML dinámico con atributos asp-*.
  • Ideal para formularios simples o páginas de contenido.

Controller CRUD (fragmento)

CRUD completo en un controller:

public class EstudiantesController : Controller
{
    public async Task<IActionResult> Index(string búsqueda)
    {
        var estudiantes = await _service.ObtenerTodosAsync();
        if (!string.IsNullOrEmpty(búsqueda))
            estudiantes = estudiantes.Where(e => e.Nombre.Contains(búsqueda));
        return View(estudiantes);
    }

    [HttpPost]
    [ValidateAntiForgeryToken]
    public async Task<IActionResult> Crear(EstudianteViewModel vm)
    {
        if (!ModelState.IsValid) return View(vm);
        await _service.CrearAsync(Mapear(vm));
        return RedirectToAction(nameof(Index));
    }
}

Pasos en Visual Studio 2022

  1. Abre Visual Studio 2022 y crea un nuevo proyecto ASP.NET Core Web App (Model-View-Controller)
  2. Nombra el proyecto EstudiantesMVC y selecciona .NET 8
  3. En Models/Estudiante.cs, crea la clase con propiedades Codigo, Nombre, CarreraId
  4. En Controllers/EstudiantesController.cs, scaffoldea el CRUD usando Entity Framework
  5. En Program.cs, añade una ruta personalizada para búsqueda: buscar/{termino}
  6. En las vistas Views/Estudiantes/*.cshtml, usa Tag Helpers: asp-for, asp-action, asp-validation-for
  7. Ejecuta y prueba: crear, editar, eliminar y buscar estudiantes

Checklist de verificación

  • ✅ El proyecto compila sin errores
  • ✅ Las vistas usan Tag Helpers (asp-*) en lugar de HTML tradicional
  • ✅ La ruta personalizada /buscar/{termino} funciona correctamente
  • ✅ Los formularios incluyen [ValidateAntiForgeryToken] en el controller
  • ✅ Las validaciones de ModelState están implementadas

  • ❌ Olvidar validación de ModelState antes de guardar
  • ❌ No proteger formularios con anti-forgery tokens
  • ❌ No usar Tag Helpers (asp-*) para vincular modelo a vista
