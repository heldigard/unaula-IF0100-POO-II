1. Que es TDD?

Test-Driven Development (TDD)

TDD es una practica de desarrollo donde se escriben las pruebas ANTES del codigo de produccion.

Por que TDD?

Beneficios:

  • Codigo mas confiable: Las pruebas garantizan funcionalidad

  • Refactorizacion segura: Las pruebas protegen contra regresiones

  2. No escribiras mas de una prueba suficiente para fallar (y compilar)
  3. No escribiras mas codigo del necesario para pasar la prueba

3. xUnit Framework

Que es xUnit?

xUnit es un framework de pruebas unitarias para .NET. Es open source y ampliamente utilizado en la comunidad .NET.

Atributos Principales

Atributo Uso Ejemplo
[Fact]Prueba sin parametrosUn caso de prueba especifico
[Theory]Prueba parametrizadaMultiple casos con datos diferentes
[InlineData(...)]Datos para [Theory]Proporciona valores para los parametros

Instalacion

# Crear proyecto de pruebas
dotnet new xunit -n MiProyecto.Tests

# Agregar referencia al proyecto principal
cd MiProyecto.Tests
dotnet add reference ../MiProyecto/MiProyecto.csproj

# Ejecutar pruebas
dotnet test

Ejemplo Completo

[Fact]
public void Sumar_DosNumeros_RetornaSuma()
{
    // Arrange - Configurar
    var calc = new Calculadora();
    int esperado = 8;

    // Act - Ejecutar
    int resultado = calc.Sumar(5, 3);

    // Assert - Verificar
    Assert.Equal(esperado, resultado);
}

5. Test Doubles

Que son los Test Doubles?

Son objetos sustitutos usados en pruebas para simular dependencias externas o complejas.

Tipos de Test Doubles

<>Llena parametros, sin uso real
Tipo Proposito Ejemplo
Dummy Objeto vacio para cumplir firma
Stub Respuestas predefinidas Siempre retorna "hola"
Fake Implementacion simplificada Repository con lista en memoria
Mock Verifica comportamiento Confirma que se llamo al metodo

Cuando usar cada uno?

Dummy

// Solo para llenar parametros
public void Probar(DummyServicio servicio) {
    // El servicio no se usa
}

Usar cuando no necesitas interactuar con el objeto.

Stub

// Respuestas predefinidas
var stub = new StubRepo();
stub.Obtener(1).Returns(new Estudiante());

// Test usa el stub
var estudiante = repo.Obtener(1);

Usar cuando necesitas respuestas controladas.

Fake

// Implementacion simplificada
public class FakeRepo : IRepositorio {
    private List _datos = new();
    public void Guardar(Estudiante e) => _datos.Add(e);
    public Estudiante Obtener(int id) => _datos.FirstOrDefault();
}

Usar cuando necesitas logica real pero simple.

Mock

// Verifica comportamiento
var mock = new Mock<IRepositorio>();
mock.Setup(x => x.Guardar(It.IsAny()))
    .Verifiable();

// Test ejecuta...
mock.Verify(); // Confirma llamada

Usar cuando necesitas verificar interacciones.

Ejemplo 2: Billetera con Validaciones

Clase Billetera

public class Billetera
{
    private decimal _saldo;

    public decimal Saldo => _saldo;

    public void Agregar(decimal monto)
    {
        if (monto <= 0)
            throw new ArgumentException("Monto debe ser positivo");
        _saldo += monto;
    }

    public void Retirar(decimal monto)
    {
        if (monto <= 0)
            throw new ArgumentException("Monto debe ser positivo");
        if (monto > _saldo)
            throw new InvalidOperationException("Saldo insuficiente");
        _saldo -= monto;
    }
}

Tests Completos

public class BilleteraTests
{
    [Fact]
    public void Saldo_Inicialmente_EsCero()
    {
        var billetera = new Billetera();
        Assert.Equal(0, billetera.Saldo);
    }

    [Fact]
    public void Agregar_MontoPosicional_IncrementaSaldo()
    {
        var billetera = new Billetera();
        billetera.Agregar(100);
        Assert.Equal(100, billetera.Saldo);
    }

    [Fact]
    public void Retirar_SaldoSuficiente_DecrementaSaldo()
    {
        var billetera = new Billetera();
        billetera.Agregar(100);
        billetera.Retirar(30);
        Assert.Equal(70, billetera.Saldo);
    }

    [Fact]
    public void Retirar_SaldoInsuficiente_LanzaExcepcion()
    {
        var billetera = new Billetera();
        billetera.Agregar(50);
        Assert.Throws<InvalidOperationException>(
            () => billetera.Retirar(100)
        );
    }
}

Ejemplo 3: Parametrizacion con [Theory]

[InlineData(1, 1, 2)]
[InlineData(5, 3, 8)]

Ejercicio 2: Validador de Contrasenas

public class ValidadorContrasenas
{
    public void Validar(string contrasena)
    {
        if (contrasena.Length < 8)
            throw new ArgumentException("Minimo 8 caracteres");
        if (!contrasena.Any(char.IsUpper))
            throw new ArgumentException("Al menos una mayuscula");
        if (!contrasena.Any(char.IsDigit))
            throw new ArgumentException("Al menos un numero");
    }
}

Requisitos:

  • Minimo 8 caracteres
  • Al menos una mayuscula
  • Al menos un numero
  • Debe lanzar excepcion si no cumple
