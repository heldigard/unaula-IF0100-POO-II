Conceptos Te√≥ricos

¬øQu√© es Pydantic?

Pydantic es una librer√≠a de Python para validaci√≥n de datos usando type hints. Es el coraz√≥n de FastAPI para:

  • Validar datos de entrada (request body)
  • Convertir datos (serializaci√≥n/deserializaci√≥n)
  • Documentar modelos autom√°ticamente (OpenAPI)
  • Proveer autocompletado en editores
Integraci√≥n FastAPI + Pydantic: FastAPI usa Pydantic para validar autom√°ticamente todo el request body, query parameters y path parameters.

Por qu√© usar Pydantic

Validaci√≥n Autom√°tica

Los datos se validan autom√°ticamente antes de que tu c√≥digo se ejecute. Errores se convierten en HTTP 422.

Documentaci√≥n Auto-Generada

Los schemas Pydantic se convierten autom√°ticamente en documentaci√≥n OpenAPI en /docs.

Type Hints

Usa type hints est√°ndar de Python para definir tipos, lo que mejora el autocompletado.

Serializaci√≥n

Convierte autom√°ticamente objetos Python a JSON y viceversa.

Usar Schemas en FastAPI

# main.py
from fastapi import FastAPI, HTTPException, status
from typing import List
from schemas.usuario import UsuarioCreate, UsuarioResponse

app = FastAPI()

# Base de datos simulada
db_usuarios = []
next_id = 1

@app.post("/usuarios", response_model=UsuarioResponse, status_code=201)
def crear_usuario(usuario: UsuarioCreate):
    """
    Crea un nuevo usuario.

    - request body: UsuarioCreate (con password)
    - response: UsuarioResponse (sin password)
    """
    global next_id

    # Verificar duplicados
    for u in db_usuarios:
        if u["username"] == usuario.username:
            raise HTTPException(
                status_code=400,
                detail="Username ya existe"
            )

    # Crear usuario
    nuevo_usuario = UsuarioResponse(
        id=next_id,
        username=usuario.username,
        email=usuario.email,
        nombre_completo=usuario.nombre_completo,
        activo=True,
        creado_en=datetime.now()
    )

    db_usuarios.append(nuevo_usuario.dict())
    next_id += 1

    return nuevo_usuario

@app.get("/usuarios", response_model=List[UsuarioResponse])
def listar_usuarios():
    """Retorna lista de usuarios SIN passwords."""
    return db_usuarios

@app.get("/usuarios/{usuario_id}", response_model=UsuarioResponse)
def obtener_usuario(usuario_id: int):
    """Retorna un usuario espec√≠fico."""
    for usuario in db_usuarios:
        if usuario["id"] == usuario_id:
            return usuario

    raise HTTPException(status_code=404, detail="Usuario no encontrado")

Validaci√≥n Personalizada

Validadores con @field_validator

from pydantic import BaseModel, field_validator, ValidationInfo

class UsuarioCreate(BaseModel):
    username: str
    email: str
    password: str
    password_confirm: str

    @field_validator('username')
    @classmethod
    def username_alphanumerico(cls, v: str) -> str:
        """Valida que username sea alfanum√©rico."""
        if not v.isalnum():
            raise ValueError('Username debe ser alfanum√©rico')
        return v

    @field_validator('password_confirm')
    @classmethod
    def passwords_coinciden(cls, v: str, info: ValidationInfo) -> str:
        """Valida que passwords coincidan."""
        if 'password' in info.data and v != info.data['password']:
            raise ValueError('Los passwords no coinciden')
        return v

    @field_validator('password')
    @classmethod
    def password_fuerte(cls, v: str) -> str:
        """Valida que el password sea fuerte."""
        if len(v) < 8:
            raise ValueError('Password muy corto (m√≠nimo 8)')
        if not any(c.isupper() for c in v):
            raise ValueError('Password debe tener al menos una may√∫scula')
        if not any(c.isdigit() for c in v):
            raise ValueError('Password debe tener al menos un n√∫mero')
        return v

from pydantic import BaseModel, field_validator, ValidationInfo, model_validator

Buenas Pr√°cticas

Organizaci√≥n de Schemas

schemas/

DO's con Pydantic

  • Usa Field para validaciones: M√°s expresivo que type hints solos
  • Crea schemas separados: Create, Update, Response
  • Nunca incluyas passwords en response: Vulnerabilidad de seguridad
  • Usa from_attributes=True: Para crear desde modelos ORM
  • Documenta con description: Aparece en /docs
  • Usa examples: Para mostrar ejemplos en documentaci√≥n

DON'T's con Pydantic

  • Problema de seguridad
  • Siempre valida datos de entrada
  • Solo validaci√≥n de datos
  • Los modelos son otra capa

    """Actualiza un usuario."""
    # Buscar usuario

        usuario_existente["nombre_completo"] = usuario.nombre_completo
    if usuario.activo is not None:
        usuario_existente["activo"] = usuario.activo

    return UsuarioResponse(**usuario_existente)
