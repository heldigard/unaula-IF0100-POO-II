Teoria

Patron MVC y responsabilidades

El controlador orquesta el flujo, el modelo contiene reglas y datos, y la vista presenta HTML. La separacion evita mezclar UI con logica de negocio.

Controller: recibe request, valida, decide flujo
Model: entidades, reglas, acceso a datos
View: presentacion HTML/CSS (Razor)

Routing y atributos HTTP

Model Binding y ViewModels

ASP.NET Core mapea datos HTTP a objetos C#. Los ViewModels separan el modelo de dominio de la UI e incluyen validaciones.

public class EstudianteViewModel
{
    [Required]
    [Display(Name = "Codigo")]
    public string Codigo { get; set; }

    [Required]
    public string Nombre { get; set; }

    public SelectList Carreras { get; set; }
}

Razor Pages y Tag Helpers

  • Razor Pages simplifica paginas con PageModel + .cshtml.
  • Tag Helpers generan HTML dinamico con atributos asp-*.
  • Ideal para formularios simples o paginas de contenido.

Controller CRUD (fragmento)

public class EstudiantesController : Controller
{
    public async Task<IActionResult> Index(string busqueda)
    {
        var estudiantes = await _service.ObtenerTodosAsync();
        if (!string.IsNullOrEmpty(busqueda))
            estudiantes = estudiantes.Where(e => e.Nombre.Contains(busqueda));
        return View(estudiantes);
    }

    [HttpPost]
    [ValidateAntiForgeryToken]
    public async Task<IActionResult> Crear(EstudianteViewModel vm)
    {
        if (!ModelState.IsValid) return View(vm);
        await _service.CrearAsync(Mapear(vm));
        return RedirectToAction(nameof(Index));
    }
}

Checklist de errores comunes

  • Colocar HTML en el controller.
  • Usar el modelo de dominio directamente en la vista.
  • Olvidar validacion de ModelState.
  • No proteger formularios con anti-forgery.
