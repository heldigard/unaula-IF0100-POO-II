¿Por qué SQLAlchemy? (10 min)

En el desarrollo profesional con Python, SQLAlchemy es el ORM estándar. Nos permite:

  • Trabajar con bases de datos usando objetos Python en lugar de SQL crudo
  • Cambiar entre SQLite (dev) y PostgreSQL/MySQL (prod) sin tocar el código
  • Aprovechar la POO: herencia, polimorfismo y encapsulamiento en nuestros modelos
Stack del curso: SQLAlchemy 2.0 + SQLite para desarrollo local. En producción puedes migrar a PostgreSQL sin cambiar una línea de tus modelos.

Modelos Declarativos (30 min)

SQLAlchemy 2.0 usa el estilo declarativo. Todos los modelos heredan de una clase base (DeclarativeBase), aplicando el concepto de herencia de POO.

1. Configurar la base

from sqlalchemy import create_engine, String, Integer, Boolean, ForeignKey
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column, relationship, Session
from typing import List, Optional

class Base(DeclarativeBase):
    """Clase base para todos los modelos del proyecto."""
    pass

# Motor SQLite (archivo local)
engine = create_engine("sqlite:///taskflow.db", echo=True)

2. Modelo Usuario

class Usuario(Base):
    __tablename__ = "usuarios"

    id: Mapped[int] = mapped_column(primary_key=True)
    username: Mapped[str] = mapped_column(String(50), unique=True, nullable=False)
    email: Mapped[str] = mapped_column(String(100), unique=True, nullable=False)
    hashed_password: Mapped[str] = mapped_column(String(255), nullable=False)
    activo: Mapped[bool] = mapped_column(Boolean, default=True)

    # Relación: un usuario tiene muchos proyectos
    proyectos: Mapped[List["Proyecto"]] = relationship(back_populates="usuario")

    def __repr__(self) -> str:
        return f""

3. Modelo Proyecto

class Proyecto(Base):
    __tablename__ = "proyectos"

    id: Mapped[int] = mapped_column(primary_key=True)
    nombre: Mapped[str] = mapped_column(String(100), nullable=False)
    descripcion: Mapped[Optional[str]] = mapped_column(String(255), nullable=True)

    # Clave foránea
    usuario_id: Mapped[int] = mapped_column(ForeignKey("usuarios.id"))

    # Relaciones
    usuario: Mapped["Usuario"] = relationship(back_populates="proyectos")
    tareas: Mapped[List["Tarea"]] = relationship(back_populates="proyecto")

    def __repr__(self) -> str:
        return f""

4. Crear las tablas

# Crear todas las tablas definidas en los modelos que heredan de Base
Base.metadata.create_all(engine)
OOP en acción: Todos los modelos (Usuario, Proyecto, Tarea) heredan de Base. Esto les da automáticamente la capacidad de mapearse a tablas SQL. Es herencia aplicada a persistencia.

Relaciones entre Entidades (20 min)

TaskFlow requiere una jerarquía clara: Usuario → Proyectos → Tareas. Esto se modela con relaciones ForeignKey y relationship.

Modelo Tarea

from enum import Enum as PyEnum
from sqlalchemy import Enum

class EstadoTarea(str, PyEnum):
    PENDIENTE = "pendiente"
    EN_PROGRESO = "en_progreso"
    COMPLETADA = "completada"

class Tarea(Base):
    __tablename__ = "tareas"

    id: Mapped[int] = mapped_column(primary_key=True)
    titulo: Mapped[str] = mapped_column(String(150), nullable=False)
    descripcion: Mapped[Optional[str]] = mapped_column(String(500), nullable=True)
    estado: Mapped[EstadoTarea] = mapped_column(
        Enum(EstadoTarea),
        default=EstadoTarea.PENDIENTE
    )

    proyecto_id: Mapped[int] = mapped_column(ForeignKey("proyectos.id"))
    proyecto: Mapped["Proyecto"] = relationship(back_populates="tareas")

    def __repr__(self) -> str:
        return f""

    def cambiar_estado(self, nuevo_estado: EstadoTarea) -> None:
        """Cambia el estado de la tarea."""
        self.estado = nuevo_estado
Relaciones en SQLAlchemy 2.0: Siempre usa Mapped[T] con relationship() para aprovechar el type checking y la autocompletación en el editor.

Resumen
