Objetivo de Aprendizaje

Implementar persistencia relacional usando SQLAlchemy 2.0 con modelos declarativos, relaciones One-to-Many y migraciones con Alembic. Los datos deben persistir en SQLite (desarrollo) o PostgreSQL/MySQL (producción).

Modalidad de Trabajo

Individual o en parejas (máximo 2 personas). Debe ser el mismo equipo de E1-E3.

Formato de entrega: Repositorio GitHub actualizado con modelos SQLAlchemy (models/ o database.py), archivo alembic.ini configurado, y al menos una migración aplicada. README.md debe incluir comando para crear/actualizar la base de datos.

Requisitos Técnicos Detallados

1. Modelos SQLAlchemy 2.0 (40%)

Descripción: Definir entidades usando el estilo declarativo moderno de SQLAlchemy.

Modelos obligatorios:
  • Usuario: id, username (unique), email (unique), hashed_password, activo
  • Proyecto: id, nombre, descripcion, usuario_id (FK)
  • Tarea: id, titulo, descripcion, estado (enum), proyecto_id (FK)
Requisitos de tipado:
  • Todas las columnas deben usar Mapped[T] con mapped_column()
  • La clase base debe heredar de DeclarativeBase
  • Incluir __repr__ en cada modelo
Ejemplo mínimo: 
class Base(DeclarativeBase):
    pass

class Usuario(Base):
    __tablename__ = "usuarios"
    id: Mapped[int] = mapped_column(primary_key=True)
    username: Mapped[str] = mapped_column(String(50), unique=True)
    email: Mapped[str] = mapped_column(String(100), unique=True)
2. Relaciones entre Entidades (30%)

Descripción: Modelar la jerarquía Usuario → Proyectos → Tareas usando relaciones SQLAlchemy.

Relaciones obligatorias:
  • Usuario.proyectos → relationship("Proyecto", back_populates="usuario")
  • Proyecto.usuario → relationship("Usuario", back_populates="proyectos")
  • Proyecto.tareas → relationship("Tarea", back_populates="proyecto")
  • Tarea.proyecto → relationship("Proyecto", back_populates="tareas")
Constraints:
  • ForeignKey("usuarios.id") en Proyecto.usuario_id
  • ForeignKey("proyectos.id") en Tarea.proyecto_id
  • Usar nullable=False en las FK cuando aplique
3. Migraciones con Alembic (30%)

Descripción: Gestionar cambios de esquema mediante migraciones versionadas.

Requisitos obligatorios:
  • Alembic inicializado en el proyecto (alembic init alembic)
  • alembic.ini configurado con URL a la base de datos
  • env.py importando Base.metadata de los modelos
  • Al menos una migración autogenerada que cree las tablas iniciales
  • Archivo de migración aplicado (alembic upgrade head)
Verificación rápida: 
alembic history --verbose
alembic current
